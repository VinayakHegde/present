Light Q&A

How do you approach frontend architecture?
Start from user journeys and data boundaries. Server state in React Query; UI state local. Shared components for repeated patterns (tables, export). Contract-first APIs so FE types stay honest. Feature flags for safe rollout.
How do you handle real-time data in the UI?
Separate transport (WebSocket/poll) from presentation. Throttle renders; show freshness timestamp; handle disconnect explicitly. Never show stale data as live — finance users notice immediately.
React vs Vue — preference?
React is my primary stack and I'm productive day one. Vue is familiar in concept — I'd ramp quickly. I care more about product outcomes and team conventions than framework religion.
How do you balance speed and quality?
Scope MVPs ruthlessly with product. Golden fixtures + tests on correctness paths. Ship behind flags; measure adoption. Calculated metrics: 1 week discovery-to-prod with zero P1 — speed with guardrails.
Mentoring example?
4-layer test template with junior FE; independent delivery on IRR + export tests; checklist embedded in squad PR template.
What distinguishes senior engineers?
Ownership beyond their PR — product outcomes, unblocking others, making good scope calls, leaving the codebase better. (Mirror this in how you answer — it's likely what they're listening for.)
